Sybiz Vision Help Redesign Prototype

This start page explains how to review the prototype. The screens are interactive: click the cards, task groups and tabs to see how the proposed help experience would work.

Start from Help Home → Open topic example

What this prototype is showing

The prototype demonstrates a proposed navigation model for Sybiz Vision Help: users can start by purpose, workflow, role or reference need, while still reaching existing F1-mapped help topics.

  • Five main entry points on the Help Home page
  • Task-based TOC groups that follow the order of work
  • Role-Based Job Aids as a shortcut layer over the same topics
  • Redesigned topic layout with clearer how-to steps and reference information

Important review tip

Some prototype elements behave like buttons even when they look like cards. Click the cards to move through the prototype, and click the task-based TOC cards to change the selected detail panel on the same page.

Recommended review path: follow the five steps below first. This gives the clearest view of the planning idea: Home entry points → Workflow guide → Task-based TOC → Existing F1 topic redesign.

Design idea to notice

These are the key planning points the prototype is intended to communicate.

Purpose-based entryUsers do not all start with the same question. The Home page separates getting started, setup, workflows, customisation and glossary needs.
Task-based TOCWorkflow sections are ordered by the order of work, not alphabetically and not only by ribbon structure.
F1 compatibilityThe workflow path does not replace existing F1 help. It gives users another way to reach the relevant topic.
Readable topic pagesTopic content is split into practical steps and reference details so users can act first and look up fields when needed.
