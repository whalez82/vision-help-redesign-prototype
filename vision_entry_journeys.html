User entry journeys

Why different users enter the manual differently

This page explains the design intent behind the Home structure. Different users come to Help with different mental models: first-use learning, screen-level help, daily workflow, role responsibility, administration or business change.

Scenario

New user learning Vision

A new user usually does not know the ribbon names or ledger terminology. They need a basic route before they can follow a transaction workflow.

Home→Getting Started with Vision→Navigation basics→Common task links
Scenario

Existing user looking for a known feature

An experienced user may already know the menu or screen name. They still need a ribbon/menu reference route.

Home→Ribbon and Menu Reference→Debtors ribbon→Sales Order
Scenario

User thinking by job role

Some users think “I am a Sales Admin” or “I work in Dispatch”, not “I need Section 19”. Job-Based Aids act as shortcuts to the right workflow topics.

Home→Job-Based Aids→Sales Admin→Quote · Order · Invoice topics
Scenario

Administrator or implementation user

Admin users need setup, rights and maintenance information before ordinary users can complete workflows.

Home→Setup and Administration→Users / roles / rights→Module defaults
Scenario

Business change or customisation need

When the business changes a form, report, custom field or process, the user is not doing a normal transaction. This is why Customisation is separated.

Home→Customisation and Extensions→Reports / forms / fields→Test output
Scenario

F1 from a Vision screen

F1 remains the fastest route when the user is already on a screen. The prototype shows how the same topic can also be reached from workflow navigation.

Vision screen→F1→Existing mapped topic→Tabbed topic card
